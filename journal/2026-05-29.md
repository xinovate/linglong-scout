### 5 服务器 Clash 代理修复

**问题**：服务器 Clash 代理失效，`git pull` 无法访问 GitHub。

**排查**：
- 原配置 `/etc/clash/config.yaml` 是 base64 订阅链接，不是有效 Clash YAML
- 订阅 URL 也返回 base64 而非 Clash 配置
- 从本地 FlClash 导出配置（`~/Library/Application Support/com.follow.clash/profiles/`）

**方案**：
- 精简配置：去掉 `anytls` 节点（原版 Clash 不支持），保留 17 个 vmess 节点
- DNS 改国内公共 DNS，解决境外 DNS 不可达
- 规则精简为仅代理 GitHub + Google，其余直连
- 切换到 `➊香港🇭🇰SVIP`（hk1.tingmo.world），原 `zhuanxian.hktsz111.xyz` 域名已失效

**Docker build 加速**：Dockerfile 加 `-i https://mirrors.aliyun.com/pypi/simple/`，build 从数分钟降到 39 秒
