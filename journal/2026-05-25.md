## 2. 服务安全加固

**问题**：服务器 redacted-server-ip 上 3 个 Docker 服务端口暴露公网无认证。

**方案**：API Key 认证，本地客户端动态 IP 无法做白名单。

**实施**：

| 服务 | 措施 | 验证 |
|------|------|------|
| SearXNG (8088) | 容器绑 127.0.0.1:8089 + nginx 反代 Bearer Token | 无 token → 403 |
| RSSHub (1200) | ACCESS_KEY 环境变量 | 无 key → 503 |
| Embedding (7997) | 已有 API_KEY Bearer Token | 已有 |

SearXNG 不原生支持 API Key，用 nginx `map` 指令匹配 `Authorization` header。

**代码改动**：
- IngestConfig 新增 `searxng_api_key` + `rsshub_access_key`
- `_searxng_search()` 发送 Bearer Token header
- `_fetch_rss_feeds()` RSSHub URL 追加 `?key=xxx`
- `fetch_rss` MCP 工具同样处理
